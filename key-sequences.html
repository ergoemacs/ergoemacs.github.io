Stopping a key sequence

By convention in most of the computing world [Esc] cancels what you are doing. This is not the typical convention in emacs. For emacs, you stop what you are doing by pressing [Ctrl]+[G], because by convention.

Ergoemacs-mode allows you to cancel a key sequence with [Esc]. However, this masks the command repeat-complex-command, which is bound to [Ctrl]+[X] [ESC] [ESC].

According to the command frequency this is not used that often.

Changing a key sequence

In vanilla emacs, you cannot undo a mis-step in a key sequence, you stop your key-sequence and retype.

In ergoemacs-mode, you can take back your last keystroke by pressing backspace (unless you already finished your key sequence).

This masks the command backward-kill-sentence [Ctrl]+[X] [Backspace].

Making Emacs key sequences easier to type

Ideally, you shouldn't have to type any combinations that include modifiers. These "key-chords" are the worst for hand health.

The best combinations are [Alt]+ keys that can be pressed with the thumbs. Even better are the key combinations where you use the left hand and right hand at the same time. This is true for the ergoemacs-mode movement commands.

Emacs key sequences, on the other hand, typically use [Ctrl]+ with almost every command.

Ergoemacs-mode tries to make key sequences much easier to type in three different ways:

  • Swapping Alt and Ctl in key sequences
  • Unchorded key sequences
  • Translating key sequences so you can use an unchorded variant.
